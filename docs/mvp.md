# Spring Miner MVP 정의

## 1. MVP 목표

기존 Spring 프로젝트 관련 파일을 업로드하면, 코드와 DB의 핵심 명명 정보를 빠르게 추출하고 화면에서 검색 및 조회할 수 있는 최소 기능 제품을 제공한다.

## 2. MVP 핵심 사용자 시나리오

### 시나리오 1. 레거시 구조 탐색

유지보수 담당자가 Java, Mapper XML, SQL 파일을 업로드한 뒤 클래스명, 메소드명, 매퍼 id, 테이블명, 컬럼명을 조회한다.

### 시나리오 2. 신규 인력 온보딩

신규 개발자가 프로젝트에서 어떤 클래스와 매퍼가 존재하는지 빠르게 확인한다.

### 시나리오 3. DB 연계 분석

개발자가 특정 테이블명과 컬럼명이 소스에 어떤 방식으로 드러나는지 확인한다.

## 3. MVP 포함 기능

### 입력

- `.java`, `.xml`, `.sql`, `.txt` 파일 업로드
- 코드 텍스트 직접 붙여넣기

### 추출

- 클래스명
- 인터페이스명
- 메소드명
- 변수명
- mapper namespace
- mapper statement id
- 테이블명
- 컬럼명

### 조회

- 유형별 결과 구분
- 결과 건수 표시
- 중복 제거
- 정렬된 목록 표시
- 검색
- 결과 유형 필터
- 문서 유형 필터
- 결과 복사
- 상세 패널
- CSV 다운로드

## 4. MVP 제외 기능

- 백엔드 저장
- 사용자 계정/권한
- 프로젝트 경로 직접 탐색
- 라인 번호 표시
- 코드 미리보기
- 연관 관계 그래프
- 히스토리 비교

## 5. MVP 화면 구성

### 화면 1. 분석 화면

- 파일 업로드 영역
- 코드 붙여넣기 영역
- 분석 실행 버튼
- 초기화 버튼

### 화면 2. 결과 화면

- 분석 대상 파일 목록
- 결과 요약 카드
- 유형별 추출 결과 목록

## 6. MVP 구현 방식 제안

### 프론트엔드

- HTML
- JavaScript
- CSS

### 분석 방식

- 초기 버전은 정규식 기반 추출
- 추후 AST 및 SQL 파서 기반 고도화 검토

## 7. MVP 완료 기준

- 업로드 또는 입력된 텍스트에서 주요 명명 항목이 추출된다.
- 사용자 화면에서 유형별 결과를 확인할 수 있다.
- 사용자가 검색, 필터, 복사를 통해 결과를 재사용할 수 있다.
- 사용자가 선택 항목의 파일명, 라인 번호, 스니펫을 확인할 수 있다.
- 내부 검토용 데모로 시연 가능한 수준의 완성도를 갖춘다.
