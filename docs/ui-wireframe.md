# Spring Miner 화면 기획서

## 1. 화면 목적

사용자가 기존 Spring 프로젝트 관련 파일을 업로드하거나 코드를 붙여넣은 뒤, 추출된 명명 정보를 유형별로 조회하고 탐색할 수 있도록 한다.

## 2. 화면 구성 원칙

- 입력과 결과를 한 화면에서 처리할 수 있어야 한다.
- 사용자는 분석 대상과 결과를 동시에 확인할 수 있어야 한다.
- 결과는 유형별로 구분되어 한눈에 파악 가능해야 한다.
- 레거시 분석 도구 특성상 검색과 필터가 빠르게 동작해야 한다.

## 3. 주요 화면

### 화면 A. 분석 메인 화면

#### 영역 1. 상단 헤더

- 서비스명
- 서비스 설명
- 현재 분석 목적 안내

#### 영역 2. 입력 패널

- 파일 업로드 영역
- 코드 직접 입력 영역
- 분석 실행 버튼
- 초기화 버튼
- 샘플 입력 버튼

#### 영역 3. 분석 요약 패널

- 분석 문서 수
- 클래스/인터페이스 수
- 메소드 수
- 테이블/컬럼 수
- 분석 대상 파일 목록

#### 영역 4. 결과 패널

- 클래스명 목록
- 메소드명 목록
- 변수명 목록
- 매퍼명 목록
- 네임스페이스 목록
- 테이블명 목록
- 컬럼명 목록

## 4. 사용자 흐름

### 기본 흐름

1. 사용자가 파일을 업로드하거나 코드를 붙여넣는다.
2. 사용자가 `분석 실행` 버튼을 클릭한다.
3. 시스템이 명명 정보를 추출한다.
4. 시스템이 요약 정보와 결과 목록을 표시한다.
5. 사용자가 필요한 항목을 탐색한다.

### 보조 흐름

1. 사용자가 `샘플 채우기` 버튼을 누른다.
2. 예제 데이터가 입력된다.
3. 즉시 데모 확인이 가능하다.

### 초기화 흐름

1. 사용자가 `초기화` 버튼을 누른다.
2. 입력 데이터와 결과 화면이 모두 비워진다.

## 5. 화면별 상세 요소

### 5.1 입력 패널

#### 목적

분석 대상을 손쉽게 넣을 수 있도록 한다.

#### 상세 요소

- 다중 파일 업로드
- 허용 확장자 안내
- 대용량 텍스트 입력 가능 영역
- 사용자가 어떤 방식으로 입력해도 분석 가능

### 5.2 요약 패널

#### 목적

현재 분석 결과의 규모를 빠르게 파악하도록 한다.

#### 상세 요소

- 카드형 수치 표시
- 분석 대상 파일 목록 노출
- 분석 성공 여부를 시각적으로 확인 가능

### 5.3 결과 패널

#### 목적

추출된 명명 정보를 카테고리별로 빠르게 탐색하도록 한다.

#### 상세 요소

- 카테고리명
- 결과 개수
- 결과 목록
- 빈 결과 상태 메시지

## 6. 차기 버전 UI 확장안

### 필터 바

- 파일 유형 필터
- 결과 유형 필터
- 키워드 검색

### 상세 패널

- 선택 항목의 원본 파일 경로
- 라인 번호
- 코드 일부 미리보기
- 연관 클래스/매퍼/테이블 정보

### 다운로드 및 복사

- 결과 CSV 다운로드
- 목록 복사

## 7. 와이어프레임 텍스트 초안

```text
+----------------------------------------------------------------------------------+
| Spring Miner                                                                    |
| Spring 코드와 DB 명명 정보를 한 화면에서 추출                                   |
+----------------------------------------------------------------------------------+
| 입력 패널                 | 요약 패널                    | 결과 패널              |
|--------------------------|-----------------------------|------------------------|
| 파일 업로드              | 분석 문서 수                 | 클래스명               |
| 코드 붙여넣기            | 클래스/인터페이스 수         | 메소드명               |
| 분석 실행                | 메소드 수                    | 변수명                 |
| 샘플 채우기              | 테이블/컬럼 수               | 매퍼명                 |
| 초기화                   | 분석 대상 파일 목록          | 네임스페이스           |
|                          |                             | 테이블명               |
|                          |                             | 컬럼명                 |
+----------------------------------------------------------------------------------+
```

## 8. 구현 반영 상태

- 현재 프로토타입은 본 화면 기획의 기본 구조를 이미 반영하고 있다.
- 검색, 상세 패널, 다운로드 기능은 차기 작업 항목으로 분리한다.
