# Spring Miner 의사결정 로그

## 기록 원칙

- 범위, 기술, 일정, 구조에 영향을 주는 결정만 기록한다.
- 결정 사유와 대안을 짧게 남긴다.

## 결정 내역

### D-001

- 일자: 2026-03-25
- 주제: 초기 입력 방식
- 결정: MVP는 파일 업로드와 텍스트 붙여넣기 중심으로 시작한다.
- 사유: 현재 실제 레거시 프로젝트 연결 환경이 없고, 정적 프로토타입으로 빠르게 검증하기 적합하다.
- 대안: 프로젝트 폴더 직접 스캔
- 후속: 차기 버전에서 로컬 스캔 또는 백엔드 연동 검토

### D-002

- 일자: 2026-03-25
- 주제: 분석 방식
- 결정: MVP는 정규식 기반 추출을 사용한다.
- 사유: 구현 속도가 빠르고 개념 검증에 충분하다.
- 대안: AST 파서 및 SQL 파서 도입
- 후속: 정확도 요구가 높아지면 고도화한다.

### D-003

- 일자: 2026-03-25
- 주제: 우선 분석 범위
- 결정: Java, Mapper XML, SQL을 우선 범위로 한다.
- 사유: Spring 레거시 구조 파악에 가장 직접적인 자산이다.
- 대안: JSP, properties, yml까지 초기 포함
- 후속: 사용자 요구에 따라 확장 여부 판단
