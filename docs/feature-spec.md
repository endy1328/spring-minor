# Spring Miner 기능 상세 명세서

## 1. 문서 목적

본 문서는 `Spring Miner`의 MVP 및 차기 확장 기능을 화면 단위와 기능 단위로 구체화하기 위한 상세 명세서이다.

## 2. 기능 구조

### 대분류

- 입력 관리
- 분석 실행
- 결과 조회
- 결과 활용
- 운영 및 확장

## 3. 기능 상세

### F-01. 파일 업로드

#### 목적

사용자가 분석 대상 파일을 직접 업로드할 수 있도록 한다.

#### 설명

- 다중 파일 업로드를 지원한다.
- 폴더 업로드를 지원한다.
- 허용 파일 형식은 `.java`, `.xml`, `.sql`, `.txt`이다.
- 업로드된 파일 목록은 화면에 표시된다.

#### 입력

- 파일 객체 목록
- 폴더 내 파일 객체 목록

#### 출력

- 분석 대기 상태의 파일 목록
- 폴더 업로드 시 상대 경로 포함 문서 목록

#### 예외

- 허용되지 않은 확장자는 제외한다.
- 빈 파일은 경고 후 제외할 수 있다.

### F-02. 텍스트 직접 입력

#### 목적

파일 없이도 일부 코드나 SQL 문장을 직접 붙여넣어 분석할 수 있도록 한다.

#### 설명

- 사용자는 텍스트 영역에 Java, XML, SQL을 붙여넣을 수 있다.
- 텍스트는 업로드 파일과 함께 분석할 수 있다.

#### 입력

- 사용자 입력 텍스트

#### 출력

- 분석 대상 문서 1건으로 처리

### F-03. 분석 실행

#### 목적

입력된 파일 및 텍스트에서 명명 정보를 추출한다.

#### 설명

- 버튼 클릭 시 분석을 시작한다.
- 각 문서 유형에 따라 정규식 기반 분석 규칙을 적용한다.
- 결과는 유형별 집합으로 누적된다.
- 중복은 제거한다.

#### 처리 규칙

- Java: 클래스명, 인터페이스명, 메소드명, 변수명 추출
- XML: mapper namespace, statement id 추출
- SQL: 테이블명, 컬럼명 추출

### F-04. 결과 요약

#### 목적

분석 결과의 규모를 빠르게 확인할 수 있도록 한다.

#### 설명

- 분석 문서 수 표시
- 클래스/인터페이스 수 표시
- 메소드 수 표시
- 테이블/컬럼 수 표시

### F-05. 유형별 결과 조회

#### 목적

추출된 결과를 카테고리별로 나누어 확인할 수 있도록 한다.

#### 설명

- 클래스명
- 메소드명
- 변수명
- 매퍼명
- 네임스페이스
- 테이블명
- 컬럼명

#### 처리 규칙

- 결과는 오름차순 정렬한다.
- 중복을 제거한다.
- 값이 없으면 빈 상태 메시지를 표시한다.

### F-06. 검색

#### 목적

결과 목록에서 특정 명칭을 빠르게 찾을 수 있도록 한다.

#### 설명

- 전체 검색 또는 카테고리 내 검색을 지원한다.
- 부분 일치 검색을 제공한다.
- 대소문자 차이 없이 검색 가능해야 한다.

### F-07. 필터

#### 목적

사용자가 원하는 유형만 좁혀서 볼 수 있도록 한다.

#### 설명

- 파일 유형별 필터
- 결과 유형별 필터
- 필요 시 패키지명 또는 namespace 기준 필터

#### 현재 구현 범위

- 결과 유형별 필터
- 문서 유형별 필터

### F-08. 상세 정보 조회

#### 목적

선택된 결과가 어느 파일에서 왔는지 확인하도록 한다.

#### 설명

- 파일명
- 상대 경로
- 라인 번호
- 코드 일부 미리보기

#### 현재 구현 범위

- 파일명 표시
- 상대 경로 표시
- 라인 번호 표시
- 1줄 스니펫 미리보기

#### 제약

- 브라우저 업로드 방식에서는 실제 절대 로컬 파일 경로를 안정적으로 얻을 수 없다.
- 폴더 업로드 시에는 상대 경로만 표시할 수 있다.
- 전체 절대 경로가 필요하면 프로젝트 디렉터리 직접 스캔 기능이 추가되어야 한다.

### F-09. 결과 복사

#### 목적

추출 결과를 다른 문서나 툴에서 활용할 수 있도록 한다.

#### 설명

- 카테고리별 복사
- 전체 결과 복사
- 현재 검색/필터 상태 기준 복사

### F-10. 결과 다운로드

#### 목적

분석 결과를 외부 파일로 내보낼 수 있도록 한다.

#### 설명

- CSV 다운로드
- 추후 Excel 다운로드 확장 가능

#### 현재 구현 범위

- 현재 검색/필터 상태 기준 CSV 다운로드

### F-11. 분석 결과 스냅샷 저장

#### 목적

현재 분석 결과를 다시 활용하거나 이후 비교 기능의 기준점으로 남길 수 있도록 한다.

#### 설명

- 현재 화면 결과를 스냅샷으로 저장한다.
- 저장된 스냅샷 목록을 조회한다.
- 저장된 스냅샷을 다시 화면에 불러온다.
- 스냅샷 JSON을 복사한다.
- 스냅샷을 삭제한다.

#### 현재 구현 범위

- 브라우저 `localStorage` 기반 저장
- 최근 20개 스냅샷 유지

### F-12. 변경 이력 비교

#### 목적

저장된 분석 결과와 현재 결과의 차이를 빠르게 파악할 수 있도록 한다.

#### 설명

- 저장된 스냅샷과 현재 결과를 비교한다.
- 카테고리별 추가 항목과 제거 항목을 보여준다.

#### 현재 구현 범위

- 스냅샷 1건과 현재 화면 결과 비교
- 클래스, 메소드, 변수, 매퍼, 네임스페이스, 테이블, 컬럼 기준 diff 표시

### F-13. 연관 관계 표시

#### 목적

선택한 항목과 함께 등장하는 관련 식별자를 빠르게 파악할 수 있도록 한다.

#### 설명

- 선택 항목과 같은 문서 또는 상대 경로에 등장한 다른 카테고리 항목을 묶어서 보여준다.

#### 현재 구현 범위

- 상세 패널에서 같은 `sourcePath` 기준의 연관 클래스, 메소드, 변수, 매퍼, 네임스페이스, 테이블, 컬럼 표시

### F-14. AST 기반 Java 정밀 분석

#### 목적

정규식 기반 Java 추출의 오탐과 누락을 줄인다.

#### 설명

- 로컬 스캔 CLI에서 Java 파일은 `javac` 기반 AST로 파싱한다.
- 클래스, 메소드, 변수명을 AST에서 추출한다.
- XML, SQL은 기존 규칙을 유지한다.

#### 현재 구현 범위

- `scan-project.mjs`에서 `.java` 파일을 `JavaAstScanner.java`로 분석
- AST 결과를 기존 JSON 스키마로 병합
