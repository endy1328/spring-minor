# Spring Miner 사용자 매뉴얼

## 1. 개요

`Spring Miner`는 기존 Spring 기반 프로젝트의 코드와 DB 관련 자산에서 주요 명명 정보를 추출하고 검색할 수 있는 내부 도구다.

추출 대상 예시:

- 클래스명
- 메소드명
- 변수명
- 매퍼명
- 네임스페이스
- 테이블명
- 컬럼명

## 2. 실행 방법

브라우저에서 아래 파일을 연다.

- [index.html](/home/u24/projects/spring_miner/index.html)

## 3. 입력 방법

### 방법 1. 파일 업로드

- `.java`, `.xml`, `.sql`, `.txt` 파일을 업로드한다.

### 방법 2. 프로젝트 폴더 업로드

- 프로젝트 폴더를 업로드하면 상대 경로 기준으로 결과를 볼 수 있다.

### 방법 3. 코드 붙여넣기

- Java, XML, SQL 내용을 직접 붙여넣는다.

### 방법 4. 스캔 결과 JSON 업로드

- `scan-project.mjs`가 생성한 JSON 결과 파일을 업로드한다.

## 4. 분석 방법

1. 입력 데이터를 넣는다.
2. `분석 실행` 버튼을 누른다.
3. 요약 카드와 결과 목록을 확인한다.

## 5. 결과 탐색

### 검색

- 상단 검색창에서 이름을 검색한다.

### 결과 유형 필터

- 클래스명, 메소드명, 변수명, 매퍼명, 네임스페이스, 테이블명, 컬럼명 단위로 필터링한다.

### 문서 유형 필터

- Java, XML, SQL, TXT, Mixed 단위로 필터링한다.

### 상세 패널

- 결과 칩을 클릭하면 파일명, 상대 경로, 라인 번호, 스니펫을 확인할 수 있다.
- 같은 문서에 함께 등장한 연관 항목도 확인할 수 있다.

## 6. 결과 활용

### 복사

- 현재 결과 복사
- 전체 결과 복사
- 카테고리별 복사

### CSV 다운로드

- 현재 검색/필터 상태 기준으로 CSV 파일을 저장한다.

### 스냅샷 저장

- 현재 결과를 스냅샷으로 저장한다.
- 저장된 스냅샷을 다시 불러올 수 있다.
- JSON 복사 및 삭제도 가능하다.

### 결과 비교

- 저장된 스냅샷과 현재 결과를 비교해 추가/제거 항목을 확인한다.

## 7. 로컬 스캔 CLI

실제 프로젝트 디렉터리를 직접 스캔하려면 아래 명령을 사용한다.

```bash
node /home/u24/projects/spring_miner/scripts/scan-project.mjs /path/to/spring-project > spring-miner-report.json
```

생성된 JSON은 다시 UI에 업로드할 수 있다.

## 8. 주의사항

- 브라우저 UI의 Java 추출은 정규식 기반이다.
- 로컬 스캔 CLI의 Java 추출은 AST 기반이다.
- 폴더 업로드 시 상대 경로만 확인할 수 있다.
- 절대 경로는 브라우저 보안 제약으로 표시되지 않는다.
