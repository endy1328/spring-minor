# Spring Miner 테스트 시나리오

## 1. 문서 목적

MVP 기준으로 주요 기능의 수동 검증 항목을 정의한다.

## 2. 테스트 범위

- 파일 업로드
- 텍스트 입력
- 분석 실행
- 결과 요약
- 유형별 결과 표시
- 예외 상황 처리

## 3. 테스트 케이스

### TC-01 파일 업로드 분석

- 목적: 업로드 파일에서 명명 정보가 정상 추출되는지 확인
- 사전조건: `.java`, `.xml`, `.sql` 샘플 파일 준비
- 절차:
1. 파일 업로드
2. 분석 실행
3. 결과 확인
- 기대결과:
  - 파일 목록이 표시된다.
  - 클래스명, 메소드명, namespace, 테이블명, 컬럼명이 추출된다.

### TC-02 텍스트 직접 입력 분석

- 목적: 붙여넣은 텍스트에서 분석이 가능한지 확인
- 절차:
1. 텍스트 입력 영역에 샘플 코드 붙여넣기
2. 분석 실행
3. 결과 확인
- 기대결과:
  - 수동 입력 문서가 분석 대상에 포함된다.
  - 결과 카드가 갱신된다.

### TC-03 파일과 텍스트 혼합 분석

- 목적: 업로드 파일과 텍스트가 함께 분석되는지 확인
- 절차:
1. 파일 업로드
2. 텍스트 입력
3. 분석 실행
- 기대결과:
  - 두 입력이 모두 반영된다.

### TC-04 빈 입력 예외 처리

- 목적: 아무 입력 없이 분석 실행 시 적절히 안내하는지 확인
- 절차:
1. 아무 값도 넣지 않는다.
2. 분석 실행 클릭
- 기대결과:
  - 경고 메시지가 표시된다.
  - 결과 화면은 유지된다.

### TC-05 초기화 기능

- 목적: 입력값과 결과가 모두 초기화되는지 확인
- 절차:
1. 샘플 또는 파일로 분석 수행
2. 초기화 클릭
- 기대결과:
  - 업로드 파일이 제거된다.
  - 텍스트 입력값이 제거된다.
  - 결과가 초기 상태로 돌아간다.

### TC-06 SQL 컬럼 추출

- 목적: SQL 문장에서 컬럼명이 적절히 추출되는지 확인
- 절차:
1. `SELECT` 문과 `CREATE TABLE` 문 입력
2. 분석 실행
- 기대결과:
  - 컬럼명이 중복 없이 출력된다.

### TC-07 Mapper 정보 추출

- 목적: Mapper XML에서 namespace와 statement id가 추출되는지 확인
- 절차:
1. mapper XML 입력
2. 분석 실행
- 기대결과:
  - namespace가 표시된다.
  - select/insert/update/delete id가 표시된다.

## 4. 테스트 데이터 권장 구성

- Java 서비스 클래스 1건
- Java 컨트롤러 클래스 1건
- Mapper XML 1건
- CREATE TABLE SQL 1건
- SELECT SQL 1건

## 5. 테스트 완료 기준

- 모든 핵심 시나리오에서 오류 없이 결과가 출력된다.
- 빈 입력과 초기화 동작이 정상이다.
- 추출 결과가 최소 기대 수준을 만족한다.
