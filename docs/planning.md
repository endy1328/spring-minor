# Spring Miner 기획 문서

## 1. 개요

`Spring Framework`로 제작된 기존 시스템의 코드와 DB 관련 자산을 분석하여, 주요 명명 정보를 추출하고 검색 및 조회할 수 있는 인하우스 웹 도구를 구축한다.

이 도구는 레거시 시스템 분석 비용을 줄이고, 유지보수 및 신규 개발자의 시스템 이해를 돕는 것을 목적으로 한다.

## 2. 구축 목적

- 기존 시스템 구조 파악 시간을 단축한다.
- 코드와 DB에 사용된 명명 규칙을 일관되게 파악할 수 있도록 한다.
- 유지보수 시 참조 대상 탐색 속도를 높인다.
- 레거시 프로젝트의 구조적 자산을 내부적으로 축적한다.

## 3. 주요 수집 대상

### 코드 영역

- 패키지명
- 클래스명
- 인터페이스명
- 메소드명
- 변수명

### 매퍼 영역

- 매퍼 파일명
- `namespace`
- `select`, `insert`, `update`, `delete` id

### DB 영역

- 테이블명
- 컬럼명

### 확장 가능 항목

- DTO / VO / Entity 명
- Service / Controller / Repository 명
- URL 매핑 정보
- SQL 파라미터명
- 호출 관계

## 4. 기대 효과

- 소스 분석 자동화
- 명명 자산의 검색 가능화
- 코드와 DB 간 연결 지점 파악 용이
- 운영 및 개발 협업 시 커뮤니케이션 비용 절감

## 5. 예상 핵심 기능

1. 프로젝트 소스 업로드 또는 지정 폴더 스캔
2. Java / XML / SQL 파일 분석
3. 명명 정보 추출 및 중복 제거
4. 유형별 목록 조회
5. 검색 및 필터
6. 원본 파일 경로 및 위치 확인
7. 결과 다운로드

## 6. 기획 시 우선 결정 사항

1. 입력 방식
   프로젝트 폴더 직접 스캔 또는 파일 업로드 방식

2. 분석 범위
   Java, Mapper XML, SQL 중심으로 시작할지 여부

3. 결과 깊이
   이름 목록만 제공할지, 파일 경로와 라인 번호까지 저장할지 여부

4. 저장 방식
   일회성 분석 도구인지, 분석 결과 저장형 서비스인지 여부

5. 사용자 범위
   개발자 전용인지, 기획/운영도 사용하는지 여부

## 7. 권장 단계별 추진안

### 1차 MVP

- Java / Mapper XML / SQL 분석
- 클래스명, 메소드명, 변수명, namespace, mapper id, 테이블명, 컬럼명 추출
- 웹 화면에서 검색, 필터, 복사 제공

### 2차 확장

- 파일 경로, 라인 번호 표시
- 원본 코드 미리보기
- 클래스, 매퍼, 테이블 간 연관 관계 표시

### 3차 고도화

- 정기 스캔
- 분석 결과 저장 및 히스토리 관리
- 명명 규칙 위반 탐지
- 변경 비교 기능

## 8. 화면 구상 초안

- 상단: 프로젝트 선택, 업로드, 분석 실행
- 좌측: 유형별 필터, 파일 유형 필터
- 중앙: 추출 결과 목록
- 우측: 상세 정보, 원본 위치, 연관 객체 정보

## 9. 서비스 정의 문장

기존 Spring 기반 레거시 시스템의 Java, Mapper XML, SQL 자산을 분석하여 클래스, 메소드, 변수, 매퍼, 네임스페이스, 테이블, 컬럼 등의 명명 정보를 추출하고, 이를 검색 및 조회할 수 있는 내부 관리용 웹 도구를 구축한다.
