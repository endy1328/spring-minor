# Spring Miner 브라우저 테스트 준비 가이드

## 1. 목적

이 문서는 Codex가 `Spring Miner`를 직접 브라우저에서 테스트할 수 있도록, 실행 환경을 준비하는 방법과 확인 절차를 정리한 문서다.

## 2. 현재 제약

현재 작업 환경에서는 다음 작업은 가능하다.

- 파일 생성 및 수정
- 터미널 명령 실행
- Node, Java 기반 스크립트 실행

현재 작업 환경에서는 다음 작업은 기본적으로 불가능하다.

- 실제 브라우저 창 열기
- 화면 클릭 및 입력
- 다운로드 동작 확인
- 브라우저 콘솔 직접 확인

따라서 브라우저 자동화 도구가 설치되어 있어야 직접 UI 테스트가 가능하다.

## 3. 권장 방식

가장 권장되는 방식은 `Playwright + Chromium` 조합이다.

이유:

- 정적 HTML 테스트에 적합하다.
- 클릭, 입력, 다운로드, 스크린샷, 콘솔 로그 확인이 가능하다.
- 이후 회귀 테스트 스크립트로 확장하기 쉽다.

## 4. 준비 조건

아래 항목이 충족되면 Codex가 브라우저 테스트를 직접 진행할 수 있다.

### 필수

- `node` 실행 가능
- `npx playwright` 실행 가능 또는 프로젝트에 Playwright 설치
- `chromium` 또는 Playwright가 사용하는 브라우저 바이너리 설치

### 권장

- 정적 서버 실행 가능
- 테스트 결과 스크린샷 저장 가능
- 다운로드 폴더 접근 가능

## 5. 설치 예시

프로젝트 루트에서 아래 순서로 준비할 수 있다.

```bash
npm init -y
npm install -D playwright
npx playwright install chromium
```

## 6. 정적 서버 실행 예시

`index.html`을 직접 여는 대신 로컬 서버로 테스트하는 편이 안정적이다.

예시:

```bash
cd /home/u24/projects/spring_miner
python3 -m http.server 4173
```

이 경우 테스트 대상 URL은 다음과 같다.

```text
http://127.0.0.1:4173/index.html
```

## 7. 자동화 가능 여부 점검 체크리스트

아래 항목이 모두 충족되면 Codex가 브라우저 테스트를 직접 진행할 수 있다.

- `node -v`가 정상 동작한다.
- `npx playwright --version`이 정상 동작한다.
- `npx playwright install chromium`이 완료되어 있다.
- 정적 서버 URL에 브라우저가 접근 가능하다.
- 테스트 결과 파일을 저장할 디렉터리가 있다.

## 8. Codex가 바로 수행할 수 있는 작업

환경이 준비되면 Codex는 다음 작업을 이어서 수행할 수 있다.

1. Playwright 테스트 스크립트 작성
2. 샘플 파일 업로드 자동화
3. 검색, 필터, 상세 패널 동작 검증
4. CSV 다운로드 동작 검증
5. 스냅샷 저장/비교 기능 검증
6. 스크린샷 저장
7. 실패 시 재현 로그 정리

## 9. 권장 테스트 범위

### 기본

- 샘플 채우기 버튼
- 분석 실행
- 검색
- 결과 유형 필터
- 문서 유형 필터
- 상세 패널 표시

### 확장

- 폴더 업로드
- JSON 결과 업로드
- CSV 다운로드
- 스냅샷 저장
- 스냅샷 불러오기
- 스냅샷 비교

## 10. 사용자가 준비해주면 좋은 것

아래 중 하나가 준비되면 바로 테스트 작업으로 넘어갈 수 있다.

### 옵션 A

- Playwright 설치 완료
- Chromium 설치 완료
- 정적 서버 실행 가능

### 옵션 B

- 사용자가 브라우저에서 직접 테스트
- 오류 화면, 콘솔 로그, 스크린샷, 다운로드 결과를 전달

## 11. 다음 단계

환경 준비가 끝나면 다음 작업을 진행하면 된다.

1. Playwright 테스트 파일 생성
2. 핵심 사용자 시나리오 자동화
3. 테스트 결과를 `docs/test-report.md`에 반영
