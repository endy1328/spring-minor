# Spring Miner 일정 및 WBS

## 1. 문서 목적

본 문서는 `Spring Miner`의 MVP를 기준으로 작업 단위를 나누고, 병렬 작업이 가능하도록 실행 순서를 정의한다.

## 2. 추진 원칙

- 기획 산출물 우선 확정
- MVP 범위 집중
- 병렬 작업 가능한 항목은 에이전트 또는 담당자별로 분리
- 문서, 구현, 검증을 분리하여 진행
