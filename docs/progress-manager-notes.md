# Spring Miner 진행상황 관리 노트

## 1. 역할

이 문서는 `Spring Miner` 작업의 진행상황을 추적하고, 작업 파일과 상태를 맞추기 위한 운영 기준이다.

진행상황 관리 에이전트의 역할은 다음과 같다.

- 작업 상태를 `TODO`, `IN_PROGRESS`, `BLOCKED`, `DONE`로 구분한다.
- 작업 파일과 상태 파일이 서로 일치하는지 확인한다.
- 진행 중인 작업과 완료된 작업을 짧고 명확하게 보고한다.
- 다른 에이전트의 작업을 되돌리지 않고, 충돌 가능성을 먼저 표시한다.

## 2. 상태 라벨

### TODO

- 아직 시작하지 않은 작업
- 의존성이 정리되지 않은 작업

### IN_PROGRESS

- 현재 작업 중인 항목
- 구현, 문서화, 검증이 진행 중인 상태

### BLOCKED

- 외부 입력이 필요하거나
- 현재 정보만으로는 진행할 수 없는 상태

### DONE

- 파일 반영과 검증이 끝난 상태
- 후속 작업자가 이어받을 수 있도록 필요한 설명이 정리된 상태

## 3. 상태 동기화 규칙

- 모든 주요 작업은 별도 상태 파일에 기록한다.
- 작업 시작 시 상태를 `IN_PROGRESS`로 변경한다.
- 산출물이 파일에 반영되면 해당 항목을 `DONE`으로 변경한다.
- 막힌 이유가 생기면 `BLOCKED`와 함께 사유를 적는다.
- 작업 파일과 상태 파일은 같은 이름 또는 같은 식별자를 사용해 연결한다.

## 4. 작업 파일 동기화 방식

- 문서 작업은 `docs/` 아래 파일과 연결한다.
- 구현 작업은 실제 소스 파일 경로를 함께 적는다.
- 상태 파일에는 작업명, 담당 범위, 현재 상태, 마지막 갱신일을 반드시 포함한다.
- 진행상황 관리 에이전트는 상태 파일과 실제 파일 내용을 대조한다.
- 불일치가 있으면 상태 파일을 우선 갱신하고, 필요 시 작업 파일 주석이나 문서를 보완한다.

## 5. 보고 규칙

- 보고는 짧게 작성한다.
- 상태 변경이 있으면 변경된 파일 경로를 함께 적는다.
- 완료 보고에는 검증 방법을 한 줄로 포함한다.
- 막힘 보고에는 원인과 필요한 입력을 한 줄로 적는다.

## 6. 보고 템플릿

```text
진행 요약: [한 줄 요약]
상태: [TODO | IN_PROGRESS | BLOCKED | DONE]
변경 파일: [absolute path]
검증: [실행/확인 방법]
다음 단계: [다음에 할 일]
```

## 7. 운영 원칙

- 한 번에 하나의 우선순위 작업에 집중한다.
- 상태 파일은 실제 작업 진척과 반드시 맞춘다.
- 진행 중인 작업은 중간 결과도 기록한다.
- 다른 에이전트의 변경을 되돌리지 않는다.
- 추측으로 상태를 적지 않고, 확인 가능한 내용만 기록한다.
