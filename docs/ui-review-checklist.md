# Spring Miner UI 검토 체크리스트

## 목적

다음 UI 개선 버전에서 검색, 카테고리 필터, 복사 동작, 빈 상태, 모바일 레이아웃, 추출 결과 사용성을 빠르게 검토하기 위한 체크리스트이다.

## 검토 기준

- 기능이 동작하는지 본다.
- 사용자가 헷갈릴 지점이 없는지 본다.
- 결과를 빠르게 찾고 활용할 수 있는지 본다.
- 모바일에서도 핵심 작업이 끊기지 않는지 본다.

## 1. 검색

- 검색어 입력 후 결과가 즉시 필터링되는가.
- 대소문자 차이에 따라 누락되지 않는가.
- 부분 일치 검색이 동작하는가.
- 검색어를 지웠을 때 전체 결과로 복구되는가.
- 검색 결과가 없을 때 적절한 안내가 표시되는가.
- 검색과 카테고리 필터를 함께 적용했을 때 결과가 일관적인가.

## 2. 카테고리 필터

- 클래스, 메소드, 변수, 매퍼, 네임스페이스, 테이블, 컬럼이 명확히 구분되는가.
- 필터 변경 시 결과 개수가 즉시 갱신되는가.
- 여러 필터를 동시에 적용했을 때 동작이 예측 가능한가.
- 필터 해제 시 원래 상태로 돌아가는가.
- 활성 필터가 화면에서 분명히 보이는가.

## 3. 복사 동작

- 개별 결과를 복사할 수 있는가.
- 현재 보이는 목록 전체를 복사할 수 있는가.
- 복사 성공 여부가 사용자에게 즉시 전달되는가.
- 복사된 값이 원문과 동일한가.
- 줄바꿈이나 공백이 불필요하게 섞이지 않는가.
- 모바일에서도 복사 버튼을 누르기 쉬운가.

## 4. 빈 상태

- 입력이 없을 때 안내가 명확한가.
- 검색 결과가 없을 때 빈 상태 메시지가 적절한가.
- 카테고리별로 항목이 없을 때 혼동을 주지 않는가.
- 초기 화면이 비어 있어도 기능이 없는 것처럼 보이지 않는가.
- 빈 상태에서 다음 행동이 무엇인지 유도하는가.

## 5. 모바일 레이아웃

- 작은 화면에서 입력 영역과 결과 영역이 한 열로 자연스럽게 재배치되는가.
- 텍스트 영역과 버튼이 화면 밖으로 밀리지 않는가.
- 결과 카드가 너무 좁아지지 않는가.
- 긴 텍스트가 잘려서 읽기 어려워지지 않는가.
- 스크롤이 과도하게 중첩되지 않는가.
- 손가락으로 터치하기에 버튼 크기가 충분한가.

## 6. 추출 결과 사용성

- 결과가 유형별로 한눈에 구분되는가.
- 결과 개수가 직관적으로 보이는가.
- 중복 제거 상태가 명확한가.
- 정렬 방식이 일관적인가.
- 결과가 많을 때도 스캔하기 쉬운가.
- 결과를 실제로 재사용하기 쉬운 형태로 제공하는가.
- 이름만 보여줄 때 부족한 정보는 없는지 확인하는가.

## 7. 확인 후 기록

- 발견한 문제를 우선순위와 함께 기록한다.
- 사용성 문제와 기능 오류를 구분한다.
- 다음 작업으로 넘길 항목을 분리한다.
- 재현 조건을 함께 남긴다.

## 8. 판정

- `Pass`: 검토 기준을 충족한다.
- `Need Fix`: 사용자 혼란 또는 기능 불일치가 있다.
- `Blocked`: 데이터나 기능 부족으로 검토가 불완전하다.
